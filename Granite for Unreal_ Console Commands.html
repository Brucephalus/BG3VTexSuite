Granite adds the following console commands and variables to Unreal.

Variables

GraniteSDK.Record

Controls the recording of Granite SDK performance statistics. These can be used by Graphine to determine performance issues. Possible Values:

  • 0 (default): Don't record any Granite performance statistics.
  • 1: Record performance statistics. The statistics are saved in date-stamped log files stored in "[Unreal Project Folder]/Saved/Profiling/GraniteSDK".

GraniteSDK.DrawDebugTiles

Control the display of the tiles used internally by Granite. Possible Values:

  • 0 (default): Normal granite rendering
  • 1: Enable the tile display.

r.FastVRam.GraniteResolveBuffer

Put the GraniteResolveBuffer (an internal rendertarget used by Granite) in fast VRam or not. This allows to to fine tune fast vram on Xbox One. Used in a similar way to the existing r.FastVRam.* variables from Epic.

r.FastVRam.GraniteTranslucensyResolveBuffer

See r.FastVRam.GraniteResolveBuffer documentation.

r.FastVRam.GraniteSeparateTranslucensyResolveBuffer

See r.FastVRam.GraniteResolveBuffer documentation.

Commands

DumpGraniteSDK

This command logs a lot of info about the current state the Granites SDK objects (tilesets & caches) to the output log. It may be requested by Graphine support.
