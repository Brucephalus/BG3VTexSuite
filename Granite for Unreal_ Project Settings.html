Granite for Unreal adds a new section to the project settings. Most settings are fine by default but allow you to fine tune the performance and resource use of Granite. Note that Granite has additional per-flatform settings that appear as additional sections in the project settings window. These settings take priority over the General settings when the project runs on the platform in question. The general settings are used on desktop platforms and in the editor.

Cache Settings

This section allows you to configure the total amount of CPU and GPU memory Granite will use for CPU and GPU caches. Granite will try to divide these amounts as best as possible over the needed CPU and GPU caches. Note that the actual caches needed by Granite depends on the type of tile sets you have imported in your project, adding/removing tilesets may change the required caches.

Cache settings.

If you unhide the advanced settings (by clicking the button with the arrow below the CPU and GPU memory spinners) you can view all CPU and GPU caches needed for all tile sets. If you want to have more control over the size of eache cache you can enable advanced mode. Advanced mode allows you to override the automaticly assigned sizes of the caches, enabling fine grained control over the amount of memory used for each cache separately.

Cache settings for two CPU caches and two GPU caches.

Cpu Cache Settings

Granite shares CPU caches between tile sets with the same page size. The page size is an inherent property of a tile set and can be configured during building (see GrBuild Options Overview). For each CPU cache needed by Granite the following properties are available (and are editable in advanced mode):

  • Size: The cache size in megabytes, this is what the platform considers CPU memory (e.g. on pc this is just standard virtual memory).

Gpu Cache Settings

Granite shares GPU caches between tile set layers with the same texture format and the same tile dimensions. Again these are inherent properties of the tilesets. If you would like to get more tilesets to share the same cache you will have to ensure they match these properties.

For each GPU cache needed by Granite the following properties are available (and are editable in advanced mode):

  • Size: The cache size in megabytes
Warning
Granite lets you enter cache sizes up to 4 GB. However, this does not nececarry mean your system supports shuch big caches. Entering cache sizes which are too big for your system may cause the engine to terminate abruptly with one of the following errors:
  • Out of video memory trying to allocate a texture! Make sure your video card has the minimum required memory, try lowering the resolution and/or closing other applications that are running. Exiting....
  • Video driver crashed and was reset! Make sure your video drivers are up to date. Exiting... If you get this error after configuring large caches it may actually be simply the driver running out of memory. Unreal gives a confusing error message as it is not technically a real driver crash. Installing the latest drivers is always a good idea, but will likely not help in this case.
If you get such errors you will have to reduce the granite cache sizes or ensure there is enough video memory available through other means. (E.g. moving large textures which are not already to Granite so they don't take up additional memory.) The maximum memory available depends on your GPU but assume this is at least 0.5 GB less than the amount of video memory you have. (To accommodate for desktop, and unreal render targets and other buffers.)

Packaging Settings

Packaging Settings.
  • Directories Containing Granite Tile Sets: This lists the directories that contain Granite tile sets. Any tilesets in these directories will be copied to the output path. This is relative to the root content folder. So to scan the whole project for tilesets just enter "/". This is only needed for projects created using Granite for Unreal 4.10 or earlier and which haven't used the upgrade path! (see Gts File Management and Handling for upgrade info).

Performance Settings

Perforamnce Settings.
  • Maximum Tile Data Upload: The maximum amount of data (in bytes) Granite is allowed to upload per frame.
  • Maximum Resolves: The maximum number of resolve buffers analysed per frame by Granite.
  • Reserved Ratio: Sets the relative amount of cache memory to reserve for data which is actually visible on screen. Setting this to higher values increaes the propability of seeing popping but also allows seeing higer resolution textures while lowering this value leads to lower resolution textures with less chance of popping.
  • Disable trilinear filtering: Disables trilinear filtering for all tilesets. This will cause more visible transitions between mipmaps but offers a 20% reduction in tile size and transcoding CPU cost.

Playstation 4 Settings

These settings are used when the project runs on the Playstation 4 patform. For a detailed description of the settings please refer to the descriptions above.

Note
PS4 has less memory available than a regular desktop PC and any available memory may not always be usable due to fragmentation and aligment issues. If you have a large project and haven't configured the cache settings yet, plese check them to ensure they are within reasonable values. On PS4 1GB total for CPU and 1GB total for GPU is probably the maximum amount you can expect to use. Depending on your game content this may be significantly less.
