Texture Streaming

Table of Contents

  • Tiles

Texture streaming is the process of progressively uploading a texture to GPU memory. By progressively uploading textures, loading times can be shortened (as not all textures have to be loaded up front) and memory use can be reduced (as not all texture data needs to be present in memory at once). For example when using Granite if you have an character with 8192x8192 texture applied to it on screen, Granite may not load the backside of the Character at full resulution. Then as the virtual camera moves around the character the nececarry parts of the backside could quickly be loaded so the user won't notice this process. The advantage is that the total GPU memory can be used a lot more efficiently than with traditional textures.

Tiles

To achieve texture streaming the Granite library splits the texture in tiles. Tiles are small rectangles of texture data. Tiles are the basic unit of processing in Granite. Data is loaded, coded and processed one tile at a time. Granite also supports multiple mipmap resolution levels. To achieve this the mipmaps are split into tiles as well. It is important to note that the tiles in a Granite texture always have the same resolution. This means that on lower resolution mipmap levels there are fewer tiles rather than the same amount of smaller tiles.

Rendering with streaming textures

Since the whole texture may not be present in memory, this could lead to rendering errors if the data required by the current rendered image is not loaded. To prevent such rendering errors, Granite uses special shader code that falls back to a lower resolution version of the texture if data is not present. Using this shader is usually comparable to using a standard texture.

In the Unreal integration of Granite this is achieved in the material editor using the Granite material nodes.

Making sure the data is loaded

Another problem that arises when working with texture streaming is to determine which subset of tiles needs to be loaded to ensure a high quality rendered result. If not enough tiles are loaded the shader (see previous section) will fall back to a lower resolution version of the texture. While not significantly incorrect this may still lead to blurry and seemingly low resolution textures. On the other side, if too much data is loaded at once, a larger amount of memory may be required than is present on the user's GPU, leading to slow performance as texture data is constantly swapped in and out of GPU memory.

Granite for Unreal has automatic built-in requesting of data. Usually no special actions are required from the user to ensure data is loaded. However to get maximum quality, it may still be beneficial to provide additional hints for the texture streaming through the use of the blueprint nodes provided by Granite.
