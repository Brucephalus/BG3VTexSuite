Introduction

Tile Set Studio is the main tool for creating and editing Granite Tile Sets. Its Graphical User Interface allows the user to pack existing texture images in one or multiple Granite Tile Sets (GTS) and build the output .gts and .gtp files on disk.

Tile Set Studio can be found in your '/Binaries/x64/' folder if you use the stand-alone Granite SDK or in the Granite for Unreal Windows start menu folder (Start Menu > Programs > Granite SDK) if you use the UE4 binary version.

Building a Tile Set file using Tile Set Studio typically goes as follows:

  1. Set up a solution
  2. Set up one or more projects
  3. Add textures in various layers to the projects
  4. Build to get a Tile Set file

Projects and Solutions

Tile Set Studio allows you to create and edit multiple Granite Tile Sets in a single user interface. The user creates a 'Solution' and adds one or multiple Tile Set 'Projects' to this solution. The user can then add, edit, or remove textures from the individual projects and build specific projects or the entire solution. The screenshot above for example shows a solution loaded in Tile Set Studio with two projects.

All important information is stored in the project file, typically a 'grproj' file and the solution file, typically a '.grsln' file. The project file contains all information to build the GTS except the actual texture files (the image files). Although the project file is not meant to be edited directly by the user, it is stored in a readable XML format which can be easily checked-in to a repository.

A single project can describe multiple stacked textures (like a texture atlas). The concept of a stacked texture is explained in more detail here. In short, a stacked texture is a combination of multiple textures corresponding with a single logical asset, for example, a diffuse and normal texture for a two-layered stacked texture of a game character. The screenshot above for example shows the first project to have three stacked textures 'Glider_Granite', 'Glider_Military', and 'Glider_Salamander', with each having four layers.

GTex

Before Tile Set Studio creates a GTS, it first codes each Stacked Texture into GTex files. Think of these files as coded and tiled versions of your source textures (comparable to DDS files). You will typically have many GTex files (one per Stacked Texture), and these GTex files are combined into a single 'packed' GTS texture atlas (typically very large) from which your application streams.

Working with GTex files is a lot easier instead of working with large, monolitic GTS files. Creating GTex files takes a lot of time (reading source textures and coding them), while creating a GTS file out of GTex files is a quick process. With version control, you typically check in your project file, the source texture files, and the coded GTex files. A person that checked out these files only need to build the GTS file. Please find more info on versioning here.

Note
GTex files are not editable. You cannot change any properties after they are created. To change their properties, you need to have the source textures in place for Tile Set Studio to find them and use them to recode the GTex with the updated properties.
Although the tools allow you to build a GTS file from only the GTex files, Graphine recommends to keep your source texture files. We do not guarantee the GTex format to be stable over different release versions, so current GTex files could be incompatible with future tools. Please keep your source textures so you can rebuild your GTex files.
The build system detects the presence of GTex files. If these are up to date, they are used to construct the GTS file. This way, having GTex files present can significantly speed up GTS creation.
It is important to know that Tile Set Studio creates a build state directory (called "GrBuildState") for each project in the same folder as the project file. In this directory, cache and state files are stored allowing faster incremental builds. This folder can take up to tens of gigabytes, for example: building a 64Kx64K texture with a normal and diffuse layer can require around 36GB of storage. Building on a solid state drive can greatly increase the speed of the build process but beware of these storage requirements.

Creating new Solution and Tile Set Project

To create a Granite Tile Set, you must first create a solution. A solution holds one or more Granite Tile Set Projects. Each project will be responsible for one output GTS file. The important information is stored in a project file, typically a 'grproj' file. The project file contains all information to build the GTS except the actual texture files (the image files).

Upon creation of a solution, a first Tile Set is immediately created. The application suggests a number of predefined layer configurations. A custom layout can be chosen by selecting the Custom item in the list.

Note
  • It is not possible to add or remove layers after a project has been set up. You can only change layer types. For example, it is not possible to go from a one-layer diffuse tile set to a two-layer diffuse+normals tile set. To achieve this, you need to set up a new project with the desired layer configuration and reimport all your textures (you can do this faster by using the export to script and import from script functionality).
  • Delete a Tile Set Project by clicking the File->Delete Selected Project menu item or right mouse click on the Tile Set in the Tile Set list in the main window and click Delete Project.

Adding a Single Stacked Texture

Typically, you add and remove stacked textures (texture assets) from a project and build the project to a GTS. Stacked Textures are a set of texture images with each image corresponding to a layer, for example, a texture for diffuse and a texture for normals.

You can add one Stacked Texture at a time, or Tile Set Studio can scan directories for Stacked Textures using wildcards or patterns. Additionally, you can specify import scripts for more advanced control on Stacked Texture creation (see advanced topics).

To add a single Stacked Textures, use the menu Tile Set->Add Stacked Textures -> Single Stacked Texture. The dialog allows you to specify the texture images for each layer in the stacked texture and configure various properties of the textures. Select a source file for each layer texture and specify a name for the Stacked Texture by typing in the name in the Name field. Note that the name text field comes with a number of suggestions based on the texture names.

Note
  • UDIM textures are considered a single Stacked Texture comprising multiple source images. Hence, to add an UDIM texture, use the 'Add Single Stacked Texture' menu option.
  • Tiled textures, where a texture is represented by a number of tiled texture images are considered a single Stacked Texture comprising multiple source images. Hence, to add a tiled texture, use the 'Add Single Stacked Texture' menu option.

UDIM

Note that Tile Set Studio automatically scans to see if the selected file is part of an UDIM texture or a general tiled texture. If so, the accompanying texture files will be automatically included in the import process. To get more information on what type of files and which files Tile Set Studio has found, check the 'Info' column and 'Info' button respectively. For example, to select a UDIM texture as a source texture for a layer, choose one of the UDIM patch files (e.g., 'UdimTest1002.jpg' in the example). Tile Set Studio will automatically find the UDIM template format and add all UDIM texture files to the layer of the Stacked Texture. The same applies to a general tiled texture.

Options

The options button allows you to set the compression quality settings, image transformation (e.g., flip), and other options on a Stacked Texture basis. See the information on the stacked texture properties panel for more information on these options than given here.

  • Clear this Layer: Clear all textures for this layer.
  • Quality Level: Set the quality level for the import for the layer.
    * Flip: Set how the content for the new import should be flipped.
  • Mip map Source: Choose the source for the mip levels for the import. Tile Set Studio can either automatically generate mip maps from your content, or fetch mip maps from the source files in case a file format is used with mips present (e.g., dds file). For more information, see advanced properties.
  • Swizzle/Swizzle 0/Swizzle 1/Swizzle 2/Swizzle 3: Set the swizzle pattern to be used on the texture input for this layer. This pattern represents the mapping from source image channels to the output Tile Set channels. For more information, see advanced layer options.

Choose the source for the mip levels for the import. Tile Set Studio can either automatically generate mip maps from your content, or fetch mip maps from the source files in case a file format is used with mips present (e.g., dds file). For more information, see advanced properties.

Multiple Sources


Certain datatypes allow you to specify multiple input sources where each source will be assigned to certain channels. For example, an RGBA texture can be sourced by one RGBA texture, or by two textures, one for the RGB channels, and another for the A channel. In the 'Import Stacked Texture' dialog, look out for the option button in front of the file input control (see screenshot above). When this option button is available, you can specify multiple sources. Just click on the option button, select the packing scheme from the menu, and set the desired file input sources.

See Channel Packing for more info and an overview of which datatypes support channel packing.

Adding Multiple Stacked Textures at Once

You can add multiple Stacked Textures in a single operation by letting Tile Set Studio scan directories for Stacked Textures using wildcards or patterns. The application will try to match files for each layer to a specified filter and when matching succeeds, the source textures will be added as a single Stacked Texture.

Note
UDIM textures are considered a single Stacked Texture comprising multiple source images. Hence, to add an UDIM texture, use the 'Add Single Stacked Texture' menu option.
Tiled textures, where a texture is represented by a number of tiled texture images are considered a single Stacked Texture comprising multiple source images. Hence, to add a tiled texture, use the 'Add Single Stacked Texture' menu option.

For example, consider a directory holding the following files:

  • door_d.dds
  • door_n.dds
  • rock_d.dds
  • rock_n.dds
  • plane_d.dds

Using the filter '*_d.dds' and '*_n.dds' will result in the addition of two Stacked Textures called door and rock:

Files Match: *_d.dds, *_n.dds Stacked Texture
door_d.dds, door_n.dds OK! door
rock_d.dds, rock_n.dds OK! rock
plane_d.dds No Match, Missing *_n.dds! -

To add Stacked Textures in one batch, use the menu ‘Tile Set->Add Stacked Textures -> Batch Add. The dialog allows you to specify the texture image filters for each layer in the stacked texture and configure various properties of the textures.

The 'Options' button is similar to the one in the Add Single Stacked Texture dialog. Pressing the 'Info' button shows which files were matched by the filter.

At the bottom of the dialog form is a list of all matches over all layers that were found. In case of our example, the 'door' and 'rock' Stacked Textures would be listed here, and not the 'plane'. Use this list to inspect your import before actually adding the Stacked Textures.

The name field allows you to choose how each individual Stacked Texture will be named. The wildcard '*' in the name will be substituted by the names of the found individual files. In our example, a name 'my_asset_*' would result in Stacked Textures 'my_asset_door', and 'my_asset_rock'.

For more information on wildcards, see Wildcard and Pattern Specification.

Just like Single Stacked Textures, you can specify multiple inputs for certain datatypes. See Channel Packing for more info.

Pressing the 'Import' button will result in the addition of all found Stacked Textures to the current project.

Editing a Stacked Textures

Right mouse click on a Stacked Texture in the main view and select Edit in the context menu to edit your Stacked Texture's textures and properties. Use this dialog to edit the textures for each layer behind the Stacked Texture.

Advanced properties are available in the 'Stacked Texture Details' window. Left mouse click on a Stacked Texture in the main view to show and edit details of a Stacked Texture and its textures and layers in the Details window.

Note
Press the menu item 'Clear This Layer' of the Options button of a layer in the Edit Stacked Texture window to clear the textures of that specific layer.

Previewing and Building

Once Stacked Textures are added to the project, they can be inspected, removed, and build. Using the main view, the state of each Stacked Texture can be inspected, allowing the user to judge which changes are included in the latest build of the GTS.

To build the resulting GTS, hit the menu button 'Build-> Build Solution'. A dialog window will pop up showing the build progress and building will commence.

Note
Building creates a GrBuildState folder containing temporary files for caching and indexing. These files are used to increase build speed over successive builds. This folder can contain cache files which can have a substantial file size (e.g., up to tens of gigabytes for very large textures). To remove these intermediate build files, the user can clean a solution. Note however, that this will result in a larger build time the next time the project is build.
The build system detects the presence of GTex files. If these are up to date, they are used to construct the GTS file. This way, having GTex files present can significantly speed up GTS creation.

Cleaning a solution can be done by clicking on the 'Clean Solution' menu button in the Build menu. Rebuilding a project, this is, cleaning a project and subsequently building it, can be done using the Rebuild Solution menu button.

During build, clean, or rebuild, any progress is shown in the progress window and errors or warnings are outputted to the log list.

Advanced Tile Set Studio Topics

Custom Layer Layout and Changing Layouts

Use the 'Custom' button in the 'Add Project' dialog to set up a custom layer layout. The dialog that follows allows you to add and configure layers.

  • Add: Check this column to include the layer in the configuration
  • Name: Set a name for the layer. Note that this name cannot be changed afterwards
  • DataType: The datatype of the layer. See Datatypes for more information
  • Compression Format: The compression format of the layer. See Compression Formats for more information. Note that changing the compression format will automatically reset any individually-set quality profiles on stacked textures to the default value.
  • Default Quality: The default quality for the layer. Each Stacked Texture that does not override its quality will have this quality profile set.
  • Default Color: The default color for the layer. Empty regions in the layer will be filled with this color.

Layer types can be changed after a project has been set up as part of the configurable project settings of a Tile Set Project. Please see Project Settings .

Note
Changing the compression format for a layer will automatically reset any individually-set quality profiles on stacked textures to the default value.

Project Settings

Use the TileSet->Properties menu item or right mouse click on the Tile Set in the Tile Set list in the main window to set properties for the active Tile Set.

The following properties can be set:

Project Tab

  • Project Name: Name of the project (currently fixed)
  • Build Profile: The build profile used to code and build the Tile Set. For more information, see Build Profiles. Note that the Build Profile determines the available Compression Formats for each layer.

Layer Configuration Tab

Change the layer configuration here. Set layer datatypes, compression format, default quality, and default color.

Output Tab

  • Output Filename: Filename of the tile set to generate. The default name is the name of the project file with the .gts extension.
  • Pages folder: Set directory name where page files will be stored for the GTS. There are some restrictions to this field, for more information, see pagesdir parameter on GrBuild.

Advanced Tab

Set advanced settings.

The following checkboxes are available:

  • Fast incremental build (default on): Allow fast building by patching already built GTS files. Contact Graphine for more information.
  • High quality coding (default on): When checked, coding quality is prioritized over processing time. Depending on the compression settings, this results in two-pass encoding schemes, high quality encoder settings, etc.
  • Disable uniform coding (default off): Uniform coding allows for better coding of tiles with uniform color values. Contact Graphine for more information.
  • Disable tile redirection (default off): Tile redirection allows for better coding by detecting duplicate tiles and redirecting tiles to their duplicate counterpart. Contact Graphine for more information.
  • Disable shared mip generation (default off): By default, the highest mip levels of the GTS encompassing many stacked textures are generated every time a GTS is built, unless this option is checked. Contact Graphine for more information.
  • Enable mip stripping (default off): Mip stripping tries to remove redundancies in the stored data, resulting in a smaller .GTP file size. Disabling this option will increase file size but may avoid compatibility problems. Note that mip stripping works best in combination with the 'GTPPerGTex' paging strategy. Contact Graphine for more information.

The following options are available:

MaximumAnisotropy
The maximum desired degree of anisotropy to be used with the GTS. Using the GTS in a rendering scenario using a larger degree of anisotropy, can result in rendering artefacts.

Values
  • 0, 2, 4, 8 (default), 16
Note
  • The build tools will automatically set the correct border size for the internal tiles in order to avoid sampling artefacts when anisotropic filtering is enabled in the 3D application. For 8x anisotropic sampling, this means a border is added to each tile of 8 pixels.
  • Setting the anisotropic degree higher, results in a larger border overhead and thus increases the disk size of the GTS.
TargetDisk
Tile Set Studio can perform disk-specific optimizations. This option specifies for which disk type the Tile Set should be optimized.

Values
  • Default (default), Generic, SSD, HDD
Note
  • Set this option only when you know the Tile Set will be deployed on the specified disk types, e.g., an ssd. In doubt, set to 'Generic' to use optimizations suitable for all disk types or set to default to use the system default (which typically resorts to Generic).
Pagesize
Set the page size. This is the size of the physical byte blocks of which a GTS is comprised of that are read from disk. Within one page, up to 1024 tiles are clustered.

Values
  • unsigned int value (default: 0 (Auto))
Note
  • Set this value to a multiple of the reading buffer block size of the media holding the the Granite Tile Set file. For hard disks, for example, read operations typically occur in blocks of 4096 bytes. So set this value to be a multiple of 4096 when targeting hard disks.
  • Default value 0 will trigger an auto pagesize calculation based on minimizing the slack count, the calculated pagesize will be a multiple of 4096 bytes.
  • It is advised to experiment with the page size before releasing the final GTS as a different value might be more optimal for the required type of application and storage medium.
TileWidth
Override the default tile width. This is the width of the tiles in the GTS excluding the pixels for the tile border. The default value is 128.
Values
  • unsigned int value (default: 0 (Auto))
Note
  • The total tile width including the borders must be a multiple of 16. The total tile width can be calculated by adding twice the maximum anisotropy to the tile width (e.g. 128 + 2*8 = 144 for a maximum anisotropy level of 8)
  • The default value 0 will result in the following tile widths: Software: 128, HardwareSoftware: 512, Hardware: 512.
TileHeight
Override the default tile height. This is the height of the tiles in the GTS excluding the pixels for the tile border. The default value is 128.
Values
  • unsigned int value (default: 0 (Auto))
Note
  • The total tile height including the borders must be a multiple of 16. The total tile height can be calculated by adding twice the maximum anisotropy to the tile height (e.g. 128 + 2*8 = 144 for a maximum anisotropy level of 8)
  • The default value 0 will result in the following tile heights: Software: 128, HardwareSoftware: 256, Hardware: 256.
PagingStrategy
Override the default paging strategy. This strategy determines the GTP layout of the tileset, i.e., how many GTP files, which stacked texture resides in which GTP, which tile resides in which GTP, etc.
Values
  • GTPPerLevel (default): one GTP page file per mip level
  • GTPPerGTex: one GTP page file per GTex
Note
  • Be careful adjusting this value, the paging strategy determines where each tile ends up in each GTP file, so it has a very large impact on streaming performance. Be sure the paging strategy you choose works for your application scenario!
  • With the GTPPerGTex strategy, certain mips of a GTex are put into a shared GTP file named 'yourtileset_mips.gtp'. Keep in mind that if the GTex is relatively small, all its mips can potentially end up in the shared GTP file, and you will not see a separate GTP for this GTex.

Changing Texture Properties

Click on Stacked Textures, layers, and individual textures in the main view to change their properties.

The following settings are settable:

Stacked Texture Tab

TargetSize
Set a custom size for the stacked texture. Each layer in the texture will be resized to this custom size. Set to automatic to let the tools decide based on the size of the texture what the dimensions of the stacked texture will be.
Values
  • Automatic (default), Custom
Note
  • Tip: Use this option to resize your stacked texture to a size that is a power of two in order to be able to use a stacked texture as a tiling texture with the Clamp or Repeat Tiling Method.
Tiling
Set the stacked texture for use as a tiling texture. Set either as a non-tiling texture, a tiling texture that repeats the border pixels, or a tiling texture which clamps the border pixels.

Values
  • None, Repeat, Clamp
Note
  • This setting is only available to stacked textures which have a size that is a power of 2.
  • Unlike in previous versions of Granite, no real pixel border is added to the stacked texture. Instead, internal implicit borders are filled with clamped or repeated pixels.

Stacked Texture Layer Tab

Quality
The quality level at which the texture should be stored in the Tile Set.
Values
  • 'Default', no specific quality profile set for the layer, choice based on asset quality profile, or medium if asset quality profile is Default
  • 'Low', for lossy compression with low quality but a small disk size
  • 'Medium', for lossy compression with medium quality and a medium-sized file,
  • 'High' for lossy compression with high quality but a larger disk size,
  • 'Lossless' for lossless compression with no loss in quality, but also a larger disk size
  • 'Uncompressed' for uncompressed data, therefore with no loss in quality, but also a larger disk size
Note
  • The higher the quality, the more bits are required to store the textures in the GTS.
  • Setting the QualityProfile will define the overall quality profile for the layer, assets can override this
  • Not setting the QualityProfile will automatically result in a 'Default' setting. For more information, see Compression Options.
For more information, see Quality Settings
Flip
An optional value determining whether the content from the source file should be flipped.
Values
  • None (default)
  • 'Horizontal', Flips the image horizontally
  • 'Vertical', Flips the image vertically
  • 'Both', Flips the image both horizontally and vertically
Swizzle
An optional channel swizzle pattern. This is a pattern consisting of numbers (0,1,2,3) representing a mapping from source image channels to the output Tile Set channels.
E.g. the pattern "2103" will swizzle RGBA to BGRA. "021" will swap the green and blue channels.

Note
  • You can also use this pattern to copy channels, "0111" will copy the second channel in the third and fourth.
  • The swizzle channel designators have to be lower than the maximum amount of the number of channels in source and destination. E.g., pattern "333" is invalid for a three channel input image (should be "222").
Swizzle2/Swizzle3/Swizzle4
An optional channel swizzle pattern for the second, third, or fourth source respectively. See 'Swizzle' for more information.
InvertChannels
An optional channel invert pattern. Select the channels you want to be inverted.


Note
  • Inversion of channels occurs after swizzling.
InvertChannels2/InvertChannels3/InvertChannels4
An optional channel invert pattern for the second, third, or fourth source respectively. See 'InvertChannels' for more information.
MipSource
Choose the source for the mip levels. The import pipeline can either automatically generate mip maps from your content, or fetch mip maps from the source files in case a file format is used with mips present(e.g., dds file).

Values
  • Default (default), Generated, File
Note
  • For most assets, 'Default' resolves to 'Generated', so mips are generated from mip map level 0. For cube maps however, default resolves to 'File', so mips are fetched from the source file.
  • If 'File' was chosen, mips are read from the file. Any mips missing from the file are automatically generated using the nearest more detailed mip.
Target Size
Lower the resolution of a specific layer for a Stacked Texture by selecting the appropriate value from this list. A layer can be resized based on either the size of the source textures, or the size of the Stacked Texture.
Values
  • 'Automatic', does no resizing
  • '1/2x of Stacked Texture', '1/4...', '1/8...', resizes the layer relative to the size of the Stacked Texture
  • '1/2x of Source Texture', '1/4...', '1/8...', resizes the layer relative to the size of the source texture files
Note
  • You can use this feature to make the resolution of a layer dependent on the size of the Stacked Texture. For example, in a stacked texture with a diffuse and normal layer, you can set the size of the diffuse texture half the size of the normal layer by setting the value '1/2x of Stacked Texture' on the diffuse layer.

Quality Settings

Granite compress your textures to lower disk space requirements and improve streaming performance. Because compression may sometimes alter pixels and introduce artefacts, Granite allows you to override the quality mode for textures. A high quality mode typically results in less coding artefacts, but takes more disk space. Lossless quality profiles exist, these do not introduce coding artefacts. For more information on quality settings, see Compression Options.

Note
The quality mode applies to the textures on disk. When streaming these textures, transcoding at run time to a GPU texture format (e.g., DXT1) can introduce additional compression artefacts. For example, a texture with the quality setting 'Lossless', is compressed without artefacts on disk, yet may be transcoded to a DXT5 format in GPU textures with limited blocking artefacts.

Quality is set on the layer of a Stacked Texture. You can change the quality of a texture by selecting the Layer of a Stacked Texture in the main list view of Tile Set Studio, and selecting the desired value from the selection box in the 'Stacked Texture Details' window.

Migrating Textures to Another Tile Set

Tile Set Studio allows you to copy or move around Stacked Textures between Tile Sets of the same solution, or to Tile Sets of a different solution.
To move a Stacked Texture to another Tile Set, in the main window, drag the Stacked Textures from one Tile Set list to another. Hold Ctrl while dropping to copy it instead of moving it.

To copy or move Stacked Textures to a Tile Set belonging to a different solution, select the Stacked Textures from the main window that you would like to migrate, right-click on the selected textures, and select 'Migrate' from the context menu. A migration dialog will pop up such as the one below. Select the target solution, Tile Set, and action, and click the 'Go' button to migrate the textures.

Note
Layer configurations need to correspond between source and target Tile Sets or Tile Set Studio will show an error. For layer configurations to correspond, the Tile Sets need to hold layers with the same data type in the same order. If the layer configuration is compatible but the target Tile Set holds fewer layers than the source Tile Set, you will get a warning indicating that the copy/move operation will remove the excess layers.

Building Tile Sets for GTex Streaming

Building a Tile Set can take a long time. This is because the GTP page files of the Tile Set get optimized for best streaming results and this optimization process takes time. To speed up Tile Set building, you can opt for building a Tile Set that streams directly from GTex files, eliminating the need to build any GTP files. Instead of minutes to hours, this process typically takes seconds to minutes to complete.

You can build a Tile Set for GTex streaming by clicking on the menu Build -> Advanced -> Build Project (Solution) for GTex Streaming. For more information, please click here.
