This page gives a high-level overview of te workflow when using Granite for Unreal.

1. Create Tile Sets

Before you can use any textures with Granite texture streaming you first need to organize them in tile sets. Building tilesets is done using the Tile Set Studio. The tile sets page goes a lot more in depth on how tilesets can be created.

2. Create GraniteTileSet and GraniteTexture Objects

Once your assets are bundled in tile sets you need some way to access and reference them from within the Unreal Editor. This is is done through Granite Tile Set objects and Granite Texture objects. Granite texture objects are the equivalent of standard Unreal textures but are streamable through the Granite texture streaming system. The easiest way to create Granite Texture is by importing the .gts file you generated in the previous step using GrBuild using our Unreal import factory. This factory can simply be accessed via the standard unreal import functionality by selecting a .gts file. The factory will then create a Granite Texture object for every stacked texture that the .gts file contains.

Importing is done through the standard interface by selecting a .gts file.

3. Create/Update Materials

If you let the .gts file importer create materials for you you can immediately start to work on new materials. However, if you have existing materials you will have to adjust them for use with Granite. Once you have GraniteTexture objects you then need to refrence them in your materials. This is again very much like referencing standard Unreal textures. GraniteTextures cannot be used using standard Unreal texture nodes but require separate Granite Material Nodes . So if you are converting an existing material you may have to update it by replacing standard textures nodes with the Granite compatible equivalents. Once you got your materials converted you can just use them in your scene like any other Unreal material no other steps are nececarry to get Granite up and running.

Note
Granite will work with any Unreal meshes. However, if you use very large texture datasets (especially if they use texture coordinates outside the range 0-1 x 0-1 as is often the case with UDIM patches) it may be needed to switch on "full precision UVs" when importing your meshes in Unreal 4.
Make sure to enable full precision UVs when using very large textures.

4. Tweaking and Shipping

Finally once you got your scene up and running with Granite you may want to look in achieving the best streaming performance and resource use. Proper tweaking can greatly improve performance or save a lot of otherwise unwasted memory. Granite offers several methods of tweaking performance.

  • Project Settings: The Project Settings allow you to tweak some Global settings affecting the whole Granite subsystem.
  • Scalability Settings: The Scalability Settings hook in with the existing unreal scalability settings sytem and allow you to modify key Granite performance parameters based on the end user's preferences/hardware.
  • Blueprint Nodes: Our blueprint nodes allow you fine-grained control over streaming performance. It allows you to customize behaviour for individual textures or run-time events. For example you could use this to prioritize certain "hero" assets over other assets.
